对以下功能执行端到端测试：$ARGUMENTS

严格遵守 web-testing Skill 的测试方法论，执行完整流程：

第1步：定义验收标准
  - 用户输入什么？
  - 用户执行什么操作？
  - 用户期望看到什么结果？
  - 多长时间内应该完成？（设定上限）
  - 失败的明确标志是什么？

第2步：准备环境
  - 打开浏览器 http://localhost:3000
  - 打开DevTools的Network和Console面板
  - 刷新页面确保干净的初始状态

第3步：执行测试
  - 输入测试数据，记录T=0
  - 持续观察：T=5秒→T=10秒→T=20秒→T=30秒
  - 记录每个时间点的UI状态
  - 记录Network请求状态和响应体内容（不能只看状态码）
  - 记录Console是否有错误

第4步：判定结果（不接受模糊结论）
  - ✓ PASS：用户在合理时间内看到了完整的、正确的结果
  - ✗ FAIL：超时/错误提示/结果为空/结果不正确
  - 禁止使用模糊表述："应该没问题"、"可能会显示"、"应该快了"

第5步：重复验证
  - 至少重复3次
  - 3次都成功 = PASS
  - 任何一次失败 = 需要排查

第6步：输出测试报告
  - 测试项、输入数据、操作步骤
  - 每个时间点的观察记录
  - 最终结论（PASS/FAIL）
  - 如果FAIL：可能原因分析（前端？后端？网络？）
